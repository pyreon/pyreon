`@pyreon/query` is a Pyreon adapter for [TanStack Query](https://tanstack.com/query) core.
It returns fine-grained reactive signals instead of re-rendering entire components,
so only the parts of your UI that read `data()`, `isPending()`, or `isError()`
update when those values change.

<Callout type="info" title="Fine-grained by default">
  Every field on a query result (`data`, `error`, `isPending`, `isFetching`, etc.)
  is an independent signal. An effect that only reads `query.data()` will not re-run
  when `isFetching` toggles.
</Callout>

## Installation

```bash title="Terminal"
bun add @pyreon/query @tanstack/query-core
```

## Setup

Wrap your app root with `QueryClientProvider` to make a `QueryClient` available to
all descendant hooks.

```tsx title="src/main.tsx"
import { mount } from "@pyreon/runtime-dom"
import { h } from "@pyreon/core"
import { QueryClient, QueryClientProvider } from "@pyreon/query"
import { App } from "./App"

const queryClient = new QueryClient()

mount(
  h(QueryClientProvider, { client: queryClient },
    h(App, null),
  ),
  document.getElementById("app")!,
)
```

## `useQuery()`

Subscribe to a query with automatic caching, refetching, and stale-while-revalidate.

```tsx title="src/components/UserProfile.tsx"
import { useQuery } from "@pyreon/query"

function UserProfile(props: { userId: number }) {
  const query = useQuery(() => ({
    queryKey: ["user", props.userId],
    queryFn: () =>
      fetch(`/api/users/${props.userId}`).then(r => r.json()),
  }))

  return (
    <div>
      {() => query.isPending() ? "Loading..." : null}
      {() => query.isError() ? `Error: ${query.error()}` : null}
      {() => query.data() ? <h1>{query.data().name}</h1> : null}
    </div>
  )
}
```

### Reactive query keys

Pass a function to `useQuery()` so it can read Pyreon signals. When a signal inside
the options function changes, the observer updates and refetches automatically.

```ts
import { signal } from "@pyreon/reactivity"

const page = signal(1)

const query = useQuery(() => ({
  queryKey: ["posts", page()],
  queryFn: () => fetchPosts(page()),
}))

// Later: page.set(2) triggers a refetch with the new key
```

## `useMutation()`

Run create / update / delete operations with reactive status tracking.

```tsx title="src/components/CreatePost.tsx"
import { useMutation, useQueryClient } from "@pyreon/query"

function CreatePost() {
  const client = useQueryClient()

  const mutation = useMutation({
    mutationFn: (data: { title: string }) =>
      fetch("/api/posts", {
        method: "POST",
        body: JSON.stringify(data),
      }).then(r => r.json()),
    onSuccess: () => {
      client.invalidateQueries({ queryKey: ["posts"] })
    },
  })

  return (
    <div>
      <button
        onClick={() => mutation.mutate({ title: "New Post" })}
        disabled={() => mutation.isPending()}
      >
        {() => mutation.isPending() ? "Creating..." : "Create Post"}
      </button>
      {() => mutation.isError() ? <p>Error: {String(mutation.error())}</p> : null}
    </div>
  )
}
```

`mutate()` fires and forgets (errors are swallowed). Use `mutateAsync()` when you
need the promise:

```ts
try {
  const post = await mutation.mutateAsync({ title: "New Post" })
} catch (err) {
  // handle error
}
```

## `useInfiniteQuery()`

Paginated and infinite-scroll queries with `fetchNextPage` / `fetchPreviousPage`.

```tsx title="src/components/PostList.tsx"
import { useInfiniteQuery } from "@pyreon/query"

function PostList() {
  const query = useInfiniteQuery(() => ({
    queryKey: ["posts"],
    queryFn: ({ pageParam }) =>
      fetch(`/api/posts?cursor=${pageParam}`).then(r => r.json()),
    initialPageParam: 0,
    getNextPageParam: (lastPage) => lastPage.nextCursor,
  }))

  return (
    <div>
      {() => query.data()?.pages.map(page =>
        page.items.map(post => <p>{post.title}</p>)
      )}
      <button
        onClick={() => query.fetchNextPage()}
        disabled={() => !query.hasNextPage() || query.isFetchingNextPage()}
      >
        {() => query.isFetchingNextPage() ? "Loading more..." : "Load More"}
      </button>
    </div>
  )
}
```

## Suspense Queries

`useSuspenseQuery()` and `useSuspenseInfiniteQuery()` pair with the `QuerySuspense`
boundary component. Inside the boundary, `data` is typed as non-undefined — the
boundary guarantees children only render after the query succeeds.

```tsx title="src/components/UserCard.tsx"
import { h } from "@pyreon/core"
import { useSuspenseQuery, QuerySuspense } from "@pyreon/query"

function UserPage() {
  const user = useSuspenseQuery(() => ({
    queryKey: ["user", 1],
    queryFn: () => fetch("/api/users/1").then(r => r.json()),
  }))

  return h(QuerySuspense, {
    query: user,
    fallback: <p>Loading user...</p>,
    error: (err) => <p>Failed: {String(err)}</p>,
  },
    () => <div>
      <h1>{user.data().name}</h1>
      <p>{user.data().email}</p>
    </div>
  )
}
```

### Multiple queries

`QuerySuspense` accepts an array of queries. Children render only when all have
succeeded.

```tsx
const user = useSuspenseQuery(() => ({ queryKey: ["user"], queryFn: fetchUser }))
const posts = useSuspenseQuery(() => ({ queryKey: ["posts"], queryFn: fetchPosts }))

h(QuerySuspense, {
  query: [user, posts],
  fallback: <p>Loading...</p>,
}, () => <div>
  <h1>{user.data().name}</h1>
  <p>{posts.data().length} posts</p>
</div>)
```

## Cache Invalidation

Use `useQueryClient()` to access the query client for cache operations.

```ts
import { useQueryClient } from "@pyreon/query"

const client = useQueryClient()

// Invalidate and refetch all queries matching the key
client.invalidateQueries({ queryKey: ["posts"] })

// Remove from cache entirely
client.removeQueries({ queryKey: ["posts", 1] })

// Optimistic update
client.setQueryData(["posts", 1], (old) => ({ ...old, title: "Updated" }))
```

## Error Handling

Errors surface through the `error` and `isError` signals on every query and
mutation result. For suspense queries, unhandled errors propagate to the nearest
`QuerySuspense` error fallback or Pyreon `ErrorBoundary`.

```tsx
import { QueryErrorResetBoundary, useQueryErrorResetBoundary } from "@pyreon/query"

function App() {
  return h(QueryErrorResetBoundary, null, () => {
    const { reset } = useQueryErrorResetBoundary()
    return <div>
      <button onClick={reset}>Retry</button>
      {/* queries inside here can be retried */}
    </div>
  })
}
```
