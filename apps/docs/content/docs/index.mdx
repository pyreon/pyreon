Pyreon is a fine-grained reactivity framework for building user interfaces. It compiles to efficient
DOM operations where only the exact nodes that depend on a changed signal are updated — no virtual
DOM diffing, no component re-renders.

<Callout type="info" title="The key idea">
  Components in Pyreon run **exactly once**. All subsequent updates flow through signals directly
  to the DOM — there is no reconciler, no re-rendering, and no virtual DOM tree to diff.
</Callout>

## Why Pyreon?

| Pain point | React / Vue | Pyreon |
|---|---|---|
| Re-renders | Whole component subtree re-renders on state change | Signal updates patch only the affected DOM node |
| Stale closures | Effect deps arrays, `useCallback`, `useMemo` boilerplate | No deps arrays — tracking is automatic |
| Bundle size | React DOM ~42 kB gzip | Pyreon core + runtime-dom ~6 kB gzip |
| SSR | Full component tree hydration | Selective walk-and-claim hydration |
| Migration | — | `@pyreon/react-compat` lets you migrate file-by-file |

## Installation

```bash title="Terminal"
bun add @pyreon/core @pyreon/reactivity @pyreon/runtime-dom
```

## Quick Start

A reactive counter. Only the `<span>` node updates when the signal changes — nothing else.

```tsx title="src/main.tsx"
import { signal } from "@pyreon/reactivity"
import { mount } from "@pyreon/runtime-dom"

function Counter() {
  const count = signal(0)

  return (
    <div>
      <span>{() => count()}</span>
      <button onClick={() => count.update(n => n + 1)}>Increment</button>
    </div>
  )
}

mount(<Counter />, document.getElementById("app")!)
```

## How Reactivity Works

The render function runs once. Every dynamic value must be wrapped in `() =>` so it becomes
a reactive accessor that Pyreon can subscribe to.

```tsx
function Profile() {
  const name = signal("Alice")
  const age = signal(30)

  return (
    <div>
      {/* Static — rendered once, never updates */}
      <h1>User Profile</h1>

      {/* Reactive — updates when name changes */}
      <p>{() => name()}</p>

      {/* Reactive class — updates when age changes */}
      <span class={() => age() >= 18 ? "adult" : "minor"}>
        {() => age()} years old
      </span>
    </div>
  )
}
```

<Callout type="warn" title="Forgetting () => is the most common mistake">
  Writing `{name()}` reads the signal **once at render time** and is static. Writing
  `{() => name()}` creates a reactive accessor. When in doubt, wrap in `() =>`.
</Callout>

## Framework Comparison

<Tabs items={["Pyreon", "React", "SolidJS"]}>
<Tab value="Pyreon">
```tsx
import { signal } from "@pyreon/reactivity"

function Counter() {
  const count = signal(0)
  return (
    <button onClick={() => count.update(n => n + 1)}>
      {() => count()} clicks
    </button>
  )
}
```
**Component runs once. Signal updates patch the text node directly.**
</Tab>
<Tab value="React">
```tsx
import { useState } from "react"

function Counter() {
  const [count, setCount] = useState(0)
  return (
    <button onClick={() => setCount(n => n + 1)}>
      {count} clicks
    </button>
  )
}
```
**Component re-renders on every state change. React diffs the result.**
</Tab>
<Tab value="SolidJS">
```tsx
import { createSignal } from "solid-js"

function Counter() {
  const [count, setCount] = createSignal(0)
  return (
    <button onClick={() => setCount(n => n + 1)}>
      {count()} clicks
    </button>
  )
}
```
**Similar model to Pyreon. Solid compiles JSX; Pyreon uses a runtime renderer.**
</Tab>
