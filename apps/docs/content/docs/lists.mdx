Pyreon provides the `For` component for efficient keyed list rendering.

<Callout type="error" title="Don't use array.map() for reactive lists">
  `array.map()` inside JSX works, but it creates and destroys all DOM nodes every time
  the signal holding the array changes. `For` only patches the items that actually added,
  removed, or reordered — O(1) per item.
</Callout>

## Basic Usage

```tsx title="src/TodoList.tsx"
import { For } from "@pyreon/core"
import { signal } from "@pyreon/reactivity"

interface Todo {
  id: number
  text: string
  done: boolean
}

function TodoList() {
  const todos = signal<Todo[]>([
    { id: 1, text: "Learn Pyreon", done: false },
    { id: 2, text: "Build something", done: false },
  ])

  return (
    <ul>
      <For
        each={todos}
        key={t => t.id}
        children={t => (
          <li class={() => t().done ? "done" : ""}>{() => t().text}</li>
        )}
      />
    </ul>
  )
}
```

<Callout type="info" title="Item accessor pattern">
  The `children` function receives a **signal accessor** `t` — call `t()` to read the
  current item reactively. This means individual item properties can update without
  re-creating the DOM node.
</Callout>

## With Fallback

Show a message when the list is empty.

```tsx
<For
  each={todos}
  key={t => t.id}
  children={t => <li>{() => t().text}</li>}
  fallback={<li class="empty">No items yet</li>}
/>
```

## Reactive Items

Because `children` receives a signal accessor, you can reactively read item properties
even when only one item changes.

```tsx
function TaskList() {
  const tasks = signal<Task[]>([])

  return (
    <ul>
      <For
        each={tasks}
        key={t => t.id}
        children={task => (
          <li>
            <input
              type="checkbox"
              checked={() => task().done}
              onChange={() =>
                tasks.update(ts =>
                  ts.map(t => t.id === task().id ? { ...t, done: !t.done } : t)
                )
              }
            />
            {() => task().text}
          </li>
        )}
      />
    </ul>
  )
}
```

## `createTemplate`

For very long lists where performance is critical, `createTemplate` pre-compiles a DOM
template that is cloned for each item instead of being built from VNodes.

```ts title="src/fast-list.ts"
import { createTemplate } from "@pyreon/runtime-dom"

const ItemTemplate = createTemplate<{ text: string; done: boolean }>(
  (item) => {
    const li = document.createElement("li")
    // wire up reactivity manually for maximum performance
    return li
  }
)
```

<Callout type="info">
  `createTemplate` is an advanced API. For most lists, `For` is fast enough. Reach for
  `createTemplate` only when profiling shows list rendering is a bottleneck.
</Callout>
