---
title: "React Compatibility"
description: "React-API-compatible shims via @pyreon/react-compat for incremental migration."
---

`@pyreon/react-compat` provides React-compatible APIs built on top of Pyreon's reactivity
primitives. It lets you migrate a React codebase to Pyreon file-by-file.

<Callout type="info">
  The goal of `@pyreon/react-compat` is to make your existing React code run without changes
  during migration — not to be a permanent API. Once a component is working, migrate it
  to native Pyreon for better performance and DX.
</Callout>

## Installation

```bash title="Terminal"
bun add @pyreon/react-compat
```

## Key Differences from React

<Tabs items={["useState reads", "No deps arrays", "class not className", "One render"]}>
<Tab value="useState reads">

In React, `count` is a plain number. In Pyreon, `count` is a signal — call it to read:

```tsx
// React
const [count, setCount] = useState(0)
return <span>{count}</span>  // plain value in JSX

// Pyreon (via compat)
const [count, setCount] = useState(0)
return <span>{count()}</span>  // must call to read reactively
// or: return <span>{() => count()}</span>
```

</Tab>
<Tab value="No deps arrays">

Pyreon tracks dependencies automatically. Deps arrays are accepted but ignored:

```ts
// React — deps required
useEffect(() => {
  document.title = name
}, [name])

// Pyreon — no deps needed
useEffect(() => {
  document.title = name()  // auto-tracked
})
```

</Tab>
<Tab value="class not className">

Pyreon's JSX uses `class` (the HTML attribute). `className` is mapped automatically by
`@pyreon/react-compat`, but native Pyreon uses `class`:

```tsx
// React
<div className="container">

// Pyreon (compat maps className → class)
<div className="container">  // still works via compat

// Pyreon native
<div class="container">  // preferred
```

</Tab>
<Tab value="One render">

React components re-run on every state change. Pyreon components run once:

```tsx
// React — runs on every render
function Fetcher() {
  fetch("/api")  // called on every render!
  return <div />
}

// Pyreon — runs once (correct)
function Fetcher() {
  onMount(() => fetch("/api"))  // called once after mount
  return <div />
}
```

</Tab>
</Tabs>

## What's Not Supported

Some React APIs have no equivalent in Pyreon:

- **`useReducer`** — Use `signal` with an action pattern instead.
- **`useLayoutEffect`** — Use `onMount` with synchronous DOM reads.
- **`forwardRef`** — Pass `ref` as a plain prop (Pyreon doesn't restrict ref forwarding).
- **`useImperativeHandle`** — Return methods from the component or use a store.
- **React DevTools** — Pyreon has its own debugging approach (signal graph inspection planned).
- **React Server Components** — Not supported. Use `renderToString` / `renderToStream` instead.
