Nova components are plain functions that return a VNode. They run **exactly once**. All
reactivity happens through signals and effects, not through re-renders.

<Callout type="info" title="Three rules for Nova components">
  1. **Run once** — the component function is called once and never again.
  2. **Signals for reactive state** — use `signal()` for values that change over time.
  3. **Wrap dynamic values in `() =>`** — JSX expressions must be functions to be reactive.
</Callout>

## JSX Setup

Add to your `tsconfig.json`:

```json title="tsconfig.json"
{
  "compilerOptions": {
    "jsx": "react-jsx",
    "jsxImportSource": "@nova/core"
  }
}
```

No import needed — the JSX runtime is injected automatically.

## Writing a Component

```tsx title="src/Greeting.tsx"
import { signal } from "@nova/reactivity"

function Greeting({ name }: { name: string }) {
  const count = signal(0)

  return (
    <div>
      <h1>Hello, {name}!</h1>
      <p>You clicked <span>{() => count()}</span> times.</p>
      <button onClick={() => count.update(n => n + 1)}>Click</button>
    </div>
  )
}
```

<Callout type="warn" title="Static props vs reactive children">
  Props like `name` are plain values — they don't change. Reactive values come from
  signals. Never try to make props reactive by calling them as functions.
</Callout>

## h() Hyperscript API

JSX compiles to `h()` calls. You can also use `h()` directly without JSX.

<Tabs items={["JSX", "h() hyperscript"]}>
<Tab value="JSX">
```tsx
import { signal } from "@nova/reactivity"

function Button({ label }: { label: string }) {
  const active = signal(false)
  return (
    <button
      class={() => active() ? "btn active" : "btn"}
      onClick={() => active.update(v => !v)}
    >
      {label}
    </button>
  )
}
```
</Tab>
<Tab value="h() hyperscript">
```ts
import { h, signal } from "@nova/core"

function Button({ label }: { label: string }) {
  const active = signal(false)
  return h("button", {
    class: () => active() ? "btn active" : "btn",
    onClick: () => active.update(v => !v),
  }, label)
}
```
</Tab>
</Tabs>

## Fragment

Use `Fragment` (or `<>...</>`) to return multiple root elements.

```tsx
import { Fragment } from "@nova/core"

function List() {
  return (
    <>
      <li>First</li>
      <li>Second</li>
      <li>Third</li>
    </>
  )
}
```

## Component Composition

Components are just functions — pass them as the first argument to `h()` or use JSX tags.

```tsx title="src/App.tsx"
function Avatar({ src, alt }: { src: string; alt: string }) {
  return <img src={src} alt={alt} class="avatar" />
}

function UserCard({ user }: { user: User }) {
  return (
    <div class="card">
      <Avatar src={user.avatarUrl} alt={user.name} />
      <h2>{user.name}</h2>
    </div>
  )
}
```

## Children

Children are passed as `children` in props. They can be any `VNodeChild`.

```tsx
function Card({
  title,
  children,
}: {
  title: string
  children?: VNodeChild
}) {
  return (
    <div class="card">
      <h3>{title}</h3>
      <div class="body">{children}</div>
    </div>
  )
}

// Usage
<Card title="My Card">
  <p>Content goes here</p>
</Card>
```

## Refs

Use `ref` to get a reference to the underlying DOM element.

```tsx
import { ref, onMount } from "@nova/core"

function AutoFocus() {
  const inputRef = ref<HTMLInputElement>(null)

  onMount(() => {
    inputRef.current?.focus()
  })

  return <input ref={inputRef} type="text" />
}
```
