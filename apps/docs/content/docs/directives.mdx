---
title: "Directives"
description: "Custom directives for imperative DOM access using the n- prefix convention."
---

Directives let you run imperative logic directly on a DOM element. Any prop starting with
`n-` is treated as a directive — Pyreon calls your function with the element after it mounts,
and runs any registered cleanup functions when the element is removed.

## Built-in: n-show

The `n-show` directive toggles an element's visibility via `display: none` without unmounting it.
Pass a reactive accessor that returns a boolean:

```tsx title="src/Toggle.tsx"
import { signal } from "@pyreon/reactivity"

function Toggle() {
  const visible = signal(true)

  return (
    <div>
      <button onClick={() => visible.update(v => !v)}>Toggle</button>
      <p n-show={() => visible()}>Now you see me</p>
    </div>
  )
}
```

Unlike conditional rendering with `{() => visible() && <p>...</p>}`, `n-show` keeps the element
in the DOM and preserves its state (scroll position, form input values, etc.).

## Creating Custom Directives

A directive is a function that receives the mounted element and an `addCleanup` callback:

```ts
import type { Directive } from "@pyreon/runtime-dom"

type Directive = (el: HTMLElement, addCleanup: (fn: () => void) => void) => void
```

Any cleanup functions you register via `addCleanup` are called automatically when the element
is removed from the DOM.

### n-focus — Auto-focus on Mount

```tsx title="src/directives/focus.ts"
import type { Directive } from "@pyreon/runtime-dom"

const nFocus: Directive = (el) => {
  el.focus()
}

// Usage:
<input n-focus={nFocus} placeholder="I'm focused on mount" />
```

### n-click-outside — Detect Outside Clicks

```tsx title="src/directives/click-outside.ts"
import type { Directive } from "@pyreon/runtime-dom"

function nClickOutside(callback: () => void): Directive {
  return (el, addCleanup) => {
    const handler = (e: Event) => {
      if (!el.contains(e.target as Node)) callback()
    }
    document.addEventListener("pointerdown", handler)
    addCleanup(() => document.removeEventListener("pointerdown", handler))
  }
}

// Usage:
function Dropdown() {
  const open = signal(true)

  return (
    <div n-click-outside={nClickOutside(() => open.set(false))}>
      {() => open() && <ul class="menu">...</ul>}
    </div>
  )
}
```

### n-tooltip — Reactive Directive with Effects

Directives can use `effect()` to react to signal changes:

```tsx title="src/directives/tooltip.ts"
import type { Directive } from "@pyreon/runtime-dom"
import { effect } from "@pyreon/reactivity"

function nTooltip(text: () => string): Directive {
  return (el, addCleanup) => {
    const e = effect(() => {
      el.title = text()
    })
    addCleanup(() => e.dispose())
  }
}

// Usage:
const label = signal("Hello")

<button n-tooltip={nTooltip(() => label())}>Hover me</button>
```

## SSR Behavior

<Callout type="warn" title="Directives are skipped during SSR">
  Directives interact with the live DOM, so they are ignored during server-side rendering.
  They only run after hydration or client-side mounting.
</Callout>
