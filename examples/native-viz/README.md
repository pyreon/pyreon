# @pyreon/example-native-viz

**PRIVATE / EXPERIMENTAL.** Exhaustive multiplatform proof: ONE `src/VizApp.tsx` compiles to **web + iOS + Android**, hosting **24 web-only-rich UI surfaces** inside native `<WebView>`s with the bidirectional data bridge:

- **21 distinct ECharts chart types** (bar / horizontal / stacked · line / area / smooth · pie / doughnut / rose · scatter · effect-scatter · radar · gauge · funnel · heatmap · candlestick · sankey · graph · tree · treemap · sunburst) — `@pyreon/charts/webview`
- a **flow pipeline diagram** — `@pyreon/flow/webview`
- a **real CodeMirror 6 code editor** — `@pyreon/code/webview`
- a **real TipTap WYSIWYG** — `@pyreon/rich-text/webview`

- **Forward** — a `revenue` signal drives every data chart, and `source` / `doc` signals drive the editors; on change each surface re-renders in place (no reload).
- **Reverse** — tapping a chart bar / flow node, or editing an editor, posts back through `pyreonPostMessage` → the native `selected` label.
- All 24 share the `<WebView html={HOST} data={…} onMessage={…}>` pattern — the universal path that compiles on every target (the `<ChartWebView>` / `<FlowWebView>` / `<CodeWebView>` / `<RichTextWebView>` components are web-side sugar; their bodies can't be PMTC-lowered, but the generated host string + `<WebView>` can).

`CHART_HOST` / `FLOW_HOST` / `CODE_HOST` / `RICHTEXT_HOST` in `VizApp.tsx` are LOCAL const literals (PMTC inlines a local literal into `PyreonWebView(html:)`), generated by the host builders — regenerate with `bun scripts/gen-hosts.ts`.

- `CHART_HOST` uses the ECharts CDN; `FLOW_HOST` is fully self-contained.
- `CODE_HOST` / `RICHTEXT_HOST` load the app-bundled editor globals (`window.CM` / `window.TT`) via `<script src="./assets/{cm,tt}.js">`. CodeMirror 6 and TipTap are modular ESM (no single UMD like ECharts), so the app bundles its editor and exposes it as a global the host drives — run **`bun scripts/gen-editors.ts`** to produce those (real CodeMirror + TipTap IIFEs, gitignored). A shipping native app inlines the bundle instead — `buildCodeHostHtml({ codemirrorScript })` / `buildRichTextHostHtml({ tiptapScript })` — for an offline, App-Store-safe page.

## Build

```
bun run gen-editors    # produce assets/{cm,tt}.js (real CodeMirror + TipTap bundles) — for running the app
bun run gen-hosts      # regenerate the embedded host literals
bun run build          # → generated/{swift,kotlin}/VizApp.{swift,kt}
```

PMTC emit (**24 `PyreonWebView` calls** = 21 charts + flow + code + rich-text, **0 warnings, both targets**) is covered by `@pyreon/native-compiler` tests; the host bridges are proven end-to-end against REAL ECharts / CodeMirror / TipTap in each package's `webview.browser.test` suite.
