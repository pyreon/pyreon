Add `@pyreon/validate/mini` — a tree-shakeable, function-composition validator API (Zod-mini / Valibot shape) alongside the chainable `s.` runtime. The chainable API can't tree-shake its checks (every string-format method lives on `StringSchema`'s prototype, so any schema pulls all 17 format regexes); `mini` provides **lean base constructors** (`string()`/`number()` — type-check + a generic `.check(...)`, no format methods) plus **standalone check actions** (`minLength`, `email`, `url`, `uuid`, `minValue`, `integer`, `multipleOf`, …) that prune to exactly what you import. Beautiful DX: `string().check(email(), minLength(2))` (reads like chaining) or point-free `pipe(string(), email(), minLength(2))`. Mini schemas are Standard Schema-native (the DX helpers work on them) and produce byte-identical verdicts + issues to the chainable equivalent (parity-locked). Measured win (Vite/Rollup, published bundle): a typical 3-field schema drops ~11 KB → ~6.5 KB gz (−41%). Also adds a generic `.check(...actions)` method to every schema. The shipped chainable classes are unchanged (zero behavior change for `s.` users). Corrects two stale docs claims: the runtime is not ~3.9 KB at the published entry, and the `@pyreon/validate/checks/*` subpath never existed — use `@pyreon/validate/mini`.
