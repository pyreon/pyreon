# @pyreon/i18n

Reactive internationalization with async namespace loading, interpolation, pluralization, and fallback locale support.

## Installation

```bash
bun add @pyreon/i18n
```

## Quick Start

```ts
import { createI18n } from "@pyreon/i18n"

const i18n = createI18n({
  locale: "en",
  fallbackLocale: "en",
  messages: {
    en: {
      common: {
        hello: "Hello, {{name}}!",
        items_one: "{{count}} item",
        items_other: "{{count}} items",
      },
    },
    es: {
      common: {
        hello: "¡Hola, {{name}}!",
        items_one: "{{count}} elemento",
        items_other: "{{count}} elementos",
      },
    },
  },
})

i18n.t("hello", { name: "World" })         // "Hello, World!"
i18n.t("items", { count: 1 })              // "1 item"
i18n.t("items", { count: 5 })              // "5 items"

i18n.locale.set("es")
i18n.t("hello", { name: "Mundo" })          // "¡Hola, Mundo!"
```

## API

### `createI18n(options)`

Create a reactive i18n instance.

**Options:**

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `locale` | `string` | required | Initial locale (e.g. `"en"`) |
| `fallbackLocale` | `string` | — | Fallback when key is missing in current locale |
| `messages` | `Record<string, TranslationDictionary>` | — | Static messages keyed by locale |
| `loader` | `NamespaceLoader` | — | Async function to load namespaces on demand |
| `defaultNamespace` | `string` | `"common"` | Namespace used when no prefix in key |
| `pluralRules` | `PluralRules` | — | Custom plural rules per locale |
| `onMissingKey` | `(locale, key, namespace?) => string \| void` | — | Missing key handler |

**Returns:** `I18nInstance`

### Translation Function

#### `t(key, values?)`

Translate a key with optional interpolation values.

```ts
// Simple key (uses default namespace)
i18n.t("greeting")

// Namespaced key
i18n.t("auth:errors.invalid")

// Nested key with dot notation
i18n.t("user.profile.title")

// With interpolation
i18n.t("welcome", { name: "Alice" })

// With pluralization
i18n.t("messages", { count: 3 })
```

**Key format:** `"key"` or `"namespace:key"`. Nested keys use dots: `"errors.required"`.

**Missing keys** return the key itself (e.g. `"auth:errors.unknown"`) unless `onMissingKey` provides a custom return value.

### Reactive Locale

```ts
i18n.locale()           // "en" — read current locale
i18n.locale.set("fr")   // switch locale — all t() calls reactively update
```

The `locale` is a `Signal<string>`. Reading `t()` inside an `effect` or `computed` automatically tracks locale changes.

### Async Namespace Loading

Load translations on demand:

```ts
const i18n = createI18n({
  locale: "en",
  loader: async (locale, namespace) => {
    const res = await fetch(`/locales/${locale}/${namespace}.json`)
    return res.json()
  },
})

// Load a namespace (deduplicates concurrent calls)
await i18n.loadNamespace("auth")
await i18n.loadNamespace("auth", "es") // specific locale

i18n.isLoading()          // boolean — any namespace currently loading
i18n.loadedNamespaces()   // Set<string> — namespaces loaded for current locale
```

### Adding Messages at Runtime

```ts
i18n.addMessages("en", {
  greeting: "Hey there!",
  nested: { deep: "value" },
})

// With namespace
i18n.addMessages("en", { title: "Settings" }, "settings")
```

Messages are deep-merged — existing keys are preserved unless overwritten.

### Checking Key Existence

```ts
i18n.exists("greeting")           // true
i18n.exists("auth:login.title")   // true if loaded
i18n.exists("nonexistent")        // false
```

### Available Locales

```ts
i18n.availableLocales() // ["en", "es"] — all locales with messages
```

## Interpolation

Replace `{{key}}` placeholders in translation strings:

```ts
import { interpolate } from "@pyreon/i18n"

interpolate("Hello, {{name}}!", { name: "World" })
// "Hello, World!"

interpolate("{{ count }} items", { count: 42 })
// "42 items"
```

Whitespace inside braces is trimmed. Unmatched placeholders are left as-is.

## Pluralization

Uses [CLDR plural rules](https://cldr.unicode.org/index/cldr-spec/plural-rules) via `Intl.PluralRules`.

**Suffixes:** `_zero`, `_one`, `_two`, `_few`, `_many`, `_other`

```ts
// Messages:
{
  items_one: "{{count}} item",
  items_other: "{{count}} items",
  items_zero: "No items",
}

i18n.t("items", { count: 0 })  // "No items"
i18n.t("items", { count: 1 })  // "1 item"
i18n.t("items", { count: 5 })  // "5 items"
```

### Custom Plural Rules

For languages with complex rules (e.g. Arabic):

```ts
const i18n = createI18n({
  locale: "ar",
  pluralRules: {
    ar: (count) => {
      if (count === 0) return "zero"
      if (count === 1) return "one"
      if (count === 2) return "two"
      if (count % 100 >= 3 && count % 100 <= 10) return "few"
      if (count % 100 >= 11 && count % 100 <= 99) return "many"
      return "other"
    },
  },
  ...
})
```

```ts
import { resolvePluralCategory } from "@pyreon/i18n"

resolvePluralCategory("en", 1)   // "one"
resolvePluralCategory("en", 5)   // "other"
resolvePluralCategory("ar", 3, customRules) // "few"
```

## Fallback Locale

When a key is missing in the current locale, the fallback locale is checked:

```ts
const i18n = createI18n({
  locale: "fr",
  fallbackLocale: "en",
  messages: {
    en: { common: { greeting: "Hello" } },
    fr: { common: {} }, // missing "greeting"
  },
})

i18n.t("greeting") // "Hello" (falls back to en)
```

## Types

| Type | Description |
| --- | --- |
| `I18nInstance` | Return type of `createI18n` |
| `I18nOptions` | Configuration options |
| `TranslationDictionary` | `{ [key: string]: string \| TranslationDictionary }` |
| `TranslationMessages` | `Record<string, TranslationDictionary>` |
| `NamespaceLoader` | `(locale: string, namespace: string) => Promise<TranslationDictionary \| undefined>` |
| `InterpolationValues` | `Record<string, string \| number>` |
| `PluralRules` | `Record<string, (count: number) => string>` |

## Context Pattern

### `I18nProvider` + `useI18n()`

Provide an i18n instance to the component tree via context, then consume it in any descendant:

```tsx
import { createI18n, I18nProvider, useI18n } from "@pyreon/i18n"

const i18n = createI18n({
  locale: "en",
  messages: {
    en: { common: { greeting: "Hello, {{name}}!" } },
  },
})

function App() {
  return (
    <I18nProvider instance={i18n}>
      <Greeting />
    </I18nProvider>
  )
}

function Greeting() {
  const { t } = useI18n()
  return <p>{t("greeting", { name: "World" })}</p>
}
```

`useI18n()` throws if called outside an `I18nProvider`.

### `<Trans>` Component

For rich JSX interpolation within translations, use the `<Trans>` component. Translation strings can contain `<tag>content</tag>` syntax, and you provide JSX renderers for each tag:

```tsx
import { Trans } from "@pyreon/i18n"

// Message: "Read our <link>terms of service</link> and <bold>privacy policy</bold>"

function Legal() {
  const { t } = useI18n()
  return (
    <Trans
      t={t}
      i18nKey="legal.notice"
      components={{
        link: <a href="/terms" />,
        bold: <strong />,
      }}
    />
  )
}
// Renders: Read our <a href="/terms">terms of service</a> and <strong>privacy policy</strong>
```

The `components` prop maps tag names in the translation string to JSX elements. The content between opening and closing tags becomes the children of the mapped element.

## Devtools

Import from `@pyreon/i18n/devtools` for runtime inspection:

```ts
import {
  registerI18n,
  getActiveI18nInstances,
  getI18nInstance,
  getI18nSnapshot,
  onI18nChange,
} from "@pyreon/i18n/devtools"

registerI18n("main", i18n)          // Register an instance for inspection
getActiveI18nInstances()             // Map of all registered instances
getI18nInstance("main")              // Get a specific instance
getI18nSnapshot("main")             // { locale, loadedNamespaces, availableLocales }
onI18nChange("main", (snapshot) => {
  console.log("i18n changed:", snapshot)
}) // Returns unsubscribe function
```

## Gotchas

**`t()` is reactive.** Reading it inside an `effect` or `computed` tracks the current locale and store version. Switching locales or loading namespaces automatically triggers re-evaluation.

**Namespace loading is deduplicated.** Multiple concurrent calls to `loadNamespace("auth")` result in a single fetch.

**Messages are stored by namespace.** The default namespace is `"common"`. Pass messages as `{ common: { ... } }` or use `addMessages(locale, dict, namespace)`.
